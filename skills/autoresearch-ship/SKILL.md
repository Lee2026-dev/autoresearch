# autoresearch-ship

Squashes the entire feature branch into one clean commit, pushes, and creates a PR. You do NOT merge. You do NOT push to the default branch.

---

## Context Received

- Feature ID and feature title
- All USR titles, scopes, and AC lists for this feature (from `prd.json`)
- `gate_results.json` summary (for each USR)
- `review.md` for each USR (must all show `Overall: APPROVED`)
- `branch_name` from `state.json`
- Iteration numbers per USR

---

## Procedure

### STEP 1 — Verify preconditions

Confirm all of these before proceeding:
1. Every USR in this feature has a `gate_results.json` with `"all_passed": true`
2. Every USR in this feature has a `review.md` with `Overall: APPROVED`
3. Working tree is clean: `git status`

If any precondition fails → **stop and report to orchestrator**. Do not create a PR with failing gates or unapproved review.

### STEP 2 — Verify feature branch

```bash
branch_name="<from state.json>"
current=$(git branch --show-current)
```

If `current != branch_name` → stop and alert orchestrator. Do not proceed.

### STEP 3 — Squash all feature commits into one

Find the point where this feature branch diverged from `main`:

```bash
base=$(git merge-base HEAD main)
```

Squash all commits from `base` to `HEAD` into a single clean commit:

```bash
git reset --soft "$base"
```

Build the commit message summarising all USRs in this feature:

```
feat(<feature_id>): <feature_title>

<USR-001> [backend]: <title>
  AC: <AC 1> / <AC 2> / <AC 3>
<USR-002> [frontend]: <title>
  AC: <AC 4> / <AC 5>

Gates: all passed
Reviewer: approved
```

Commit:
```bash
git commit -m "<message above>"
```

### STEP 4 — Push

```bash
git push -u origin HEAD
```

### STEP 5 — Build PR body

```markdown
## <Feature Title>

**Feature:** <feature_id> | **Tasks:** <USR-001>, <USR-002>

### USR-001 — <title> [backend]
**Acceptance Criteria**
- [x] <AC 1>
- [x] <AC 2>

### USR-002 — <title> [frontend]
**Acceptance Criteria**
- [x] <AC 3>
- [x] <AC 4>

All AC items verified by automated tests and independent reviewer.

### Quality Gates
| USR | Scope | Gates |
|---|---|---|
| USR-001 | backend | ✅ lint ✅ typecheck ✅ test ✅ security |
| USR-002 | frontend | ✅ lint ✅ typecheck ✅ test |

---
🤖 Generated by autoresearch
```

If any USR's source is `github:<N>`, append `Closes #<N>` to the body.

### STEP 6 — Create PR

```bash
gh pr create \
  --title "feat(<feature_id>): <feature_title>" \
  --body "<PR body>" \
  --base main
```

Capture and return the PR URL to the orchestrator.

### STEP 7 — Report

```
[autoresearch:ship] PR created: <URL>
Feature <feature_id> ready for human review and merge.
USR-001 [backend] ✅  USR-002 [frontend] ✅
```

---

## Hard Rules

- **NEVER run `gh pr merge`** or any variant.
- **NEVER run `git push --force`**.
- **NEVER push directly to `main` or `master`**.
- **NEVER squash if working tree is dirty** — always check `git status` first.
- If any precondition in STEP 1 fails, stop immediately. Do not proceed.
